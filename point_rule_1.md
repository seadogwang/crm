# Loyalty 积分规则配置平台设计文档（精简版）
> **关联主文档**：基于AI的下一代全渠道忠诚度管理SaaS平台 v7.3\
> **版本**：2.0\
> **设计原则**：
>
> * **简洁直观**：去除AI对话等复杂交互，回归表单配置
>
> * **所见即所得**：配置即规则，预览即理解
>
> * **灵活可扩展**：支持阶梯循环、变量控制、多积分类型组合
## 一、整体设计
### 1.1 规则配置的核心能力
| 能力       | 说明                     | 配置方式      |
| -------- | ---------------------- | --------- |
| **触发条件** | 订单金额、商品SKU、品类、会员等级、时间等 | 表单选择 + 输入 |
| **奖励计算** | 固定值、阶梯、循环阶梯            | 表格配置 + 开关 |
| **奖励对象** | 单个积分类型 / 多个积分类型组合      | 下拉选择 / 多选 |
| **上限控制** | 单笔上限、累计上限              | 数字输入 + 开关 |
| **变量引用** | 引用已定义的变量作为条件阈值         | 下拉选择变量    |

## 二、核心配置模块详解
### 2.1 触发条件配置
触发条件支持四种来源，每种来源动态展示对应的配置控件：
| 条件来源 | 示例            | 控件                |
| ---- | ------------- | ----------------- |
| 订单字段 | 订单实付金额 ≥ 100  | 字段下拉 + 运算符 + 数值输入 |
| 商品条件 | 商品列表包含 SKU\_A | 字段下拉 + 运算符 + 多选下拉 |
| 会员字段 | 会员等级 ∈ 黄金     | 字段下拉 + 运算符 + 多选下拉 |
| 变量   | 累计购买次数 ≥ 5    | 变量下拉 + 运算符 + 数值输入 |
```text
┌─ 条件配置行 ───────────────────────────────────────────────────────────────┐
│  [订单 ▼] [实付金额 ▼] [≥ ▼] [100] [元]                    [×删除]      │
│  [商品 ▼] [SKU ▼]      [包含 ▼] [SKU_A] [SKU_B]          [×删除]      │
│  [会员 ▼] [会员等级 ▼]  [∈ ▼]   [黄金] [铂金]             [×删除]      │
│  [变量 ▼] [累计购买次数 ▼] [≥ ▼] [5] [次]                 [×删除]      │
└─────────────────────────────────────────────────────────────────────────────┘
```
**变量选择器**：下拉列表自动加载 `rule_variable_definition` 中已定义的变量，显示变量名称和当前值预览。
### 2.2 奖励计算配置
#### 2.2.1 积分类型选择
支持单积分类型或多积分类型组合：
```text
积分类型： [REWARD ▼]  [移除]
          [ACTIVITY_POINTS ▼]  [移除]
          [➕ 添加积分类型]
```
多积分类型场景示例：
* 订单金额 ≥ 100 元 → 消费积分 +10，活动积分 +5
* 规则引擎同时发放两种积分，分别记录到不同的 `account_transaction`
#### 2.2.2 计算模式
| 模式   | 说明            | 适用场景     |
| ---- | ------------- | -------- |
| 固定值  | 满足条件即赠送固定积分   | 签到、首购奖励  |
| 固定倍数 | 按金额 × 倍数      | 全场多倍积分   |
| 阶梯   | 不同金额区间不同倍数/积分 | 梯度奖励     |
| 阶梯循环 | 阶梯 + 循环扣除     | 大额订单分段计算 |
**阶梯表格配置**：
```text
┌─ 阶梯规则 ────────────────────────────────────────────────────────────────┐
│  [+ 添加区间]  [清空]                                                     │
│  ┌────────────┬────────────┬──────────────┬──────────────────────────────┐│
│  │ 下限(元)   │ 上限(元)   │ 倍数/积分值  │ 循环分段点                 ││
│  ├────────────┼────────────┼──────────────┼──────────────────────────────┤│
│  │ 0          │ 1000       │ 1.0 倍       │ ☐                          ││
│  │ 1000       │ 2000       │ 1.5 倍       │ ☐                          ││
│  │ 2000       │ 3000       │ 2.0 倍       │ ☐                          ││
│  │ 3000       │ (空)       │ 3.0 倍       │ ☑ (循环)                   ││
│  └────────────┴────────────┴──────────────┴──────────────────────────────┘│
│                                                                             │
│  说明：上限为空表示无上限；勾选循环分段点的阶梯会重复扣除。                  │
└─────────────────────────────────────────────────────────────────────────────┘
```

配置实时展示预览结果，帮助运营验证逻辑：

## 三、复杂场景配置详解
### 3.1 阶梯循环配置
阶梯循环的核心是“循环分段点”——当订单金额超过某个阈值时，该阈值会被反复扣除并计算奖励，直到剩余金额不足阈值。
**配置步骤**：
1. 在阶梯表格中设置各金额区间及对应的倍数/积分值
2. 勾选需要作为循环分段点的区间（通常选择最高区间）
3. 系统自动识别循环分段点，生成循环计算逻辑
**示例：订单金额 8500 元的计算过程**
```text
阶梯规则：
  0-1000 → 1 倍
  1000-2000 → 1.5 倍
  2000-3000 → 2 倍
  3000+ → 3 倍（循环分段点）
计算过程：
  8500 ≥ 3000 → 3000×3 = 9000，剩余 5500
  5500 ≥ 3000 → 3000×3 = 9000，剩余 2500
  2500 在 2000-3000 区间 → 2000×2 = 4000，剩余 500
  500 在 0-1000 区间 → 500×1 = 500
  总计：9000 + 9000 + 4000 + 500 = 22500 积分
```
### 3.2 变量作为条件控制
在规则中引用变量作为阈值，使规则随变量值动态变化：
```text
┌─ 条件配置 ────────────────────────────────────────────────────────────────┐
│  条件来源： [变量 ▼]  变量： [累计购买次数 ▼]  运算符： [≥ ▼]  值： [5] │
│                                                                             │
│  📌 变量说明：累计购买次数 (当前值：8)                                     │
│  该变量由系统自动计算更新，基于近365天的订单数据。                         │
└─────────────────────────────────────────────────────────────────────────────┘
```
**配置存储**：
```json
{
  "conditions": [
    {
      "type": "VARIABLE",
      "varCode": "total_order_count",
      "operator": ">=",
      "threshold": 5
    }
  ]
}
```
### 3.3 多个积分类型组合奖励
配置多个积分类型时，引擎按顺序计算并累加发放：
```text
┌─ 奖励规则 ────────────────────────────────────────────────────────────────┐
│  💡 配置多个积分类型，订单满足条件时将同时发放多种积分。                   │
│                                                                             │
│  ┌─ 积分类型 1 ──────────────────────────────────────────────────────────┐ │
│  │  类型： [REWARD ▼]  计算模式： [阶梯循环 ▼]                          │ │
│  │  阶梯规则：...                                                       │ │
│  └─────────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
│  ┌─ 积分类型 2 ──────────────────────────────────────────────────────────┐ │
│  │  类型： [PREPAY_CREDIT ▼]  计算模式： [固定倍数 ▼]  倍数： [1.0] 倍 │ │
│  └─────────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
│  ┌─ 积分类型 3 ──────────────────────────────────────────────────────────┐ │
│  │  类型： [TIER_AMOUNT ▼]  计算模式： [固定值 ▼]  积分： [10] 分      │ │
│  └─────────────────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────────────┘
```
**存储格式**：
```json
{
  "reward": {
    "types": [
      {
        "pointType": "REWARD",
        "mode": "STEP_CYCLE",
        "steps": [...]
      },
      {
        "pointType": "PREPAY_CREDIT",
        "mode": "FIXED_MULTIPLIER",
        "multiplier": 1.0
      },
      {
        "pointType": "TIER_AMOUNT",
        "mode": "FIXED_VALUE",
        "points": 10
      }
    ]
  }
}
```
## 四、配置存储格式
### 4.1 规则 `metadata` 结构
```json
{
  "ruleDescription": "618指定商品多倍积分",
  "timeRange": {
    "start": "2026-06-01T00:00:00Z",
    "end": "2026-06-30T23:59:59Z"
  },
  "conditions": [
    {
      "type": "EVENT_ATTRIBUTE",
      "field": "total_amount",
      "operator": ">=",
      "value": 100
    },
    {
      "type": "EVENT_ATTRIBUTE",
      "field": "items.sku",
      "operator": "IN",
      "value": ["SKU_A", "SKU_B"]
    },
    {
      "type": "MEMBER_ATTRIBUTE",
      "field": "tier_code",
      "operator": "IN",
      "value": ["GOLD", "PLATINUM"]
    },
    {
      "type": "VARIABLE",
      "varCode": "total_order_count",
      "operator": ">=",
      "threshold": 5
    }
  ],
  "conditionLogic": "AND",
  "reward": {
    "types": [
      {
        "pointType": "REWARD",
        "mode": "STEP_CYCLE",
        "steps": [
          { "lower": 0, "upper": 1000, "multiplier": 1.0, "isCycleThreshold": false },
          { "lower": 1000, "upper": 2000, "multiplier": 1.5, "isCycleThreshold": false },
          { "lower": 2000, "upper": 3000, "multiplier": 2.0, "isCycleThreshold": false },
          { "lower": 3000, "upper": null, "multiplier": 3.0, "isCycleThreshold": true }
        ]
      },
      {
        "pointType": "ACTIVITY_POINTS",
        "mode": "FIXED_VALUE",
        "points": 10
      }
    ]
  },
  "limits": {
    "perOrderLimit": 100,
    "accumulativeLimit": 500,
    "excessStrategy": "TRUNCATE"
  }
}
```
## 五、API 设计
| 方法   | 路径                              | 说明                |
| ---- | ------------------------------- | ----------------- |
| GET  | `/api/rules`                    | 获取规则列表            |
| GET  | `/api/rules/{ruleCode}`         | 获取规则详情            |
| POST | `/api/rules`                    | 创建规则              |
| PUT  | `/api/rules/{ruleCode}`         | 更新规则              |
| POST | `/api/rules/{ruleCode}/publish` | 发布规则              |
| POST | `/api/rules/{ruleCode}/test`    | 模拟测试规则            |
| GET  | `/api/rules/variables`          | 获取可用变量列表（供条件配置使用） |
| GET  | `/api/rules/point-types`        | 获取可用积分类型列表        |
## 六、总结
| 能力        | 配置方式                         |
| --------- | ---------------------------- |
| **阶梯循环**  | 阶梯表格 + 勾选循环分段点，系统自动生成循环逻辑    |
| **变量引用**  | 条件配置中选择“变量”来源，下拉选择已定义的变量     |
| **多积分类型** | 点击“添加积分类型”，分别配置每种积分类型的计算规则   |
| **上限控制**  | 单笔上限 + 累计上限 + 超限策略（停止/截断/缩放） |
| **预览验证**  | 输入测试订单数据，实时展示匹配过程和计算结果       |

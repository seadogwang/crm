# 规则引擎补充设计文档：AI 对话式规则配置 V4（复杂业务场景全覆盖）
> **关联主文档**：基于AI的下一代全渠道忠诚度管理SaaS平台 v7.3
> **版本**：4.0
> **设计目标**：在 V3（需求澄清与动态表单）基础上，扩展支持**俱乐部基础规则配置**和**积分活动配置**两大场景，覆盖简单规则、复杂阶梯规则、循环规则等全部业务模式，并与现有页面配置能力对齐。
***
## 一、设计背景与覆盖范围
### 1.1 现有规则配置体系
项目中已存在两套规则配置页面：
| 配置页面        | 对应数据表                                      | 说明                             |
| ----------- | ------------------------------------------ | ------------------------------ |
| **俱乐部基础规则** | `rule_definition`（`rule_category='base'`）  | 配置俱乐部的基础积分规则，如订单积分、签到积分等       |
| **积分活动配置**  | `rule_definition`（`rule_category='promo'`） | 配置营销活动规则，如618大促、指定商品多倍积分、阶梯活动等 |
### 1.2 AI 辅助配置的覆盖范围
AI 对话式规则配置需要同时支持上述两套配置，并根据用户意图自动识别进入哪种配置流程：
| 用户需求示例          | 目标配置    | AI 识别依据         |
| --------------- | ------- | --------------- |
| “配置订单满100送10积分” | 俱乐部基础规则 | 无时间限制、全场商品      |
| “618活动指定商品多倍积分” | 积分活动    | 有时间限制、指定商品      |
| “配置阶梯积分活动”      | 积分活动    | 涉及阶梯、循环、上限等复杂逻辑 |
### 1.3 “指定商品多倍积分”的两种模式
| 模式             | 说明                               | 适用场景          | 判断依据               |
| -------------- | -------------------------------- | ------------- | ------------------ |
| **整单多倍（全量多倍）** | 整笔订单（含所有商品）都按多倍积分计算，指定商品只是“参与门槛” | 品牌统一促销，无差别对待  | 用户说“指定商品参与活动，整单多倍” |
| **商品多倍（定向多倍）** | 仅指定商品本身多倍，非指定商品正常累积              | 清库存、推新品，差异化运营 | 用户说“指定商品多倍，其他正常”   |
AI 在澄清阶段**必须识别用户意图属于哪种模式**，因为两者的规则脚本和配置参数完全不同。

## 二、整体设计
### 2.1 配置场景分类
| 场景分类       | 复杂度 | 典型配置项                            | AI 辅助模式          |
| ---------- | --- | -------------------------------- | ---------------- |
| **简单规则**   | 低   | 触发条件（实体+属性+运算符+值）+ 奖励动作（积分类型+数量） | 直接对话生成           |
| **固定倍数规则** | 中   | 基础规则 + 倍数                        | 澄清 + 表单          |
| **固定积分规则** | 中   | 基础规则 + 固定积分值（如“满100送10分”）        | 澄清 + 表单          |
| **商品多倍规则** | 中高  | 商品范围 + 多倍 + 非指定商品处理              | 澄清 + 表单          |
| **阶梯规则**   | 高   | 金额区间 → 不同倍数/积分值                  | 澄清 + 阶梯表格生成      |
| **循环规则**   | 高   | 阶梯规则 + 循环模式                      | 澄清 + 阶梯表格 + 循环配置 |
| **上限控制**   | 中   | 单笔上限、累计上限、超限策略                   | 表单字段补充           |
### 2.2 配置能力矩阵（AI vs 现有页面）
| 配置能力     | 现有页面 | AI 辅助   | 说明               |
| -------- | ---- | ------- | ---------------- |
| 基础触发条件配置 | ✅    | ✅       | 实体选择、属性选择、运算符、值  |
| 积分类型选择   | ✅    | ✅       | 根据规则类型自动过滤可用积分类型 |
| 固定倍数积分   | ✅    | ✅       | 简单场景，对话直接生成      |
| 固定积分值    | ✅    | ✅       | 简单场景，对话直接生成      |
| 指定商品多倍积分 | ✅    | ✅       | 需澄清整单多倍 vs 商品多倍  |
| 阶梯积分规则   | ✅    | ⚠️ 部分支持 | 复杂阶梯需配合表单（阶梯表格）  |
| 循环扣除规则   | ✅    | ⚠️ 部分支持 | 需配合表单配置循环点       |
| 积分上限控制   | ✅    | ✅       | 单笔上限、累计上限、超限策略   |
| 规则有效期    | ✅    | ✅       | 支持长期/指定时间段       |
| 规则优先级/互斥 | ✅    | ⚠️ 需澄清  | 同组规则优先级、是否叠加     |

## 三、规则类型与配置模式
### 3.1 规则类型分类
| 规则类型       | 说明             | 典型场景           | 可用积分类型                 |
| ---------- | -------------- | -------------- | ---------------------- |
| **积分累积规则** | 用户满足条件时累积积分    | 订单积分、签到积分、行为积分 | `is_redeemable = true` |
| **等级规则**   | 用户满足条件时累积等级成长值 | 升级规则、保级规则      | `is_tier_calc = true`  |
| **积分活动规则** | 限时/限条件的特殊积分规则  | 618大促、指定商品多倍积分 | `is_redeemable = true` |
### 3.2 五种奖励模式
| 模式        | 说明           | 配置要素             | 示例                        |
| --------- | ------------ | ---------------- | ------------------------- |
| **固定倍数**  | 按固定倍数奖励积分    | 倍数、是否取整          | 2倍积分                      |
| **固定积分值** | 按固定积分值奖励     | 积分值              | 满100送10积分                 |
| **阶梯倍数**  | 按金额区间不同倍数    | 区间表（下限→上限→倍数）    | 0-1000→1倍, 1000-2000→1.5倍 |
| **阶梯固定值** | 按金额区间不同固定积分值 | 区间表（下限→上限→积分值）   | 0-1000→10分, 1000-2000→20分 |
| **阶梯循环**  | 阶梯倍数 + 循环扣除  | 阶梯表 + 循环点 + 循环顺序 | 3000以上分段循环                |
### 3.3 规则有效期
| 模式        | 说明            | 配置方式         |
| --------- | ------------- | ------------ |
| **长期生效**  | 规则无时间限制       | 不设置起始时间和结束时间 |
| **指定时间段** | 规则仅在指定时间范围内生效 | 设置起始时间和结束时间  |

## 四、AI 配置流程（完整版）
### 4.1 整体流程图

```text
用户输入需求
       │
       ▼
┌──────────────────────────────────────────────────────────────┐
│ Step 1: 意图识别                                             │
│ - 识别场景：基础规则 vs 活动规则                              │
│ - 识别规则类型：累积规则 vs 等级规则                         │
│ - 识别奖励模式：固定倍数 vs 固定积分 vs 阶梯 vs 循环        │
└──────────────────────────────────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────────────────────────┐
│ Step 2: 信息抽取与澄清（2-3轮对话）                          │
│ - 触发条件：实体→属性→运算符→值                             │
│ - 奖励模式：倍数/积分值/阶梯                                 │
│ - 指定商品：整单多倍 vs 商品多倍                            │
│ - 有效期：长期 vs 指定时间                                  │
│ - 其他：等级限制、渠道限制、上限等                          │
└──────────────────────────────────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────────────────────────┐
│ Step 3: 生成动态表单                                         │
│ - 基础信息 + 触发条件 + 奖励配置                             │
│ - 阶梯场景：生成阶梯表格                                    │
│ - 循环场景：生成循环点配置                                   │
│ - 生成规则预览（业务描述 + DRL）                            │
└──────────────────────────────────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────────────────────────┐
│ Step 4: 用户确认与保存                                       │
│ - 确认规则逻辑                                              │
│ - 保存为草稿或直接发布                                      │
└──────────────────────────────────────────────────────────────┘
```
### 4.2 核心澄清维度
| 维度         | 澄清问题（业务表达）              | 选项                      |
| ---------- | ----------------------- | ----------------------- |
| **场景识别**   | 这是一个长期生效的基础规则，还是一个限时活动？ | 基础规则 / 限时活动             |
| **规则类型**   | 这个规则是累积积分还是累积等级成长值？     | 积分累积 / 等级规则             |
| **触发条件**   | 在什么情况下触发积分奖励？           | 订单金额 / 指定商品 / 签到 / 其他行为 |
| **奖励模式**   | 奖励方式是固定倍数、固定积分值，还是阶梯奖励？ | 固定倍数 / 固定积分值 / 阶梯奖励     |
| **指定商品多倍** | 多倍积分只针对指定商品，还是整笔订单？     | 整单多倍 / 商品多倍             |
| **有效期**    | 规则长期生效还是指定时间段？          | 长期生效 / 指定时间段            |
| **积分类型**   | 使用哪种积分类型？               | 消费积分 / 预售积分 / 其他        |
| **上限控制**   | 是否设置单笔订单或活动期间的上限？       | 无上限 / 单笔上限 / 累计上限       |
### 4.3 “指定商品多倍积分”澄清场景

```text
用户：618活动指定商品多倍积分
AI：收到需求。在配置之前需要确认一个关键点：
这次活动的“多倍积分”是哪种模式？
A. 整单多倍：只要订单中包含指定商品，整笔订单都按多倍积分计算
B. 商品多倍：只有指定商品本身按多倍积分计算，其他商品正常累积（1倍）
用户：商品多倍，其他正常
AI：明白了。再确认以下信息：
1. 指定商品有哪些？（选择商品）
2. 多倍积分是多少倍？
3. 是否有最低消费门槛？
4. 活动时间是？（长期/指定时间段）
用户：...（逐步补充）
```

## 六、与现有页面的集成
### 6.1 功能对应关系
| AI 对话输出     | 对应页面     | 对应数据字段                               |
| ----------- | -------- | ------------------------------------ |
| 规则类型（基础/活动） | 规则列表 Tab | `rule_category`                      |
| 规则名称/描述     | 基础信息     | `rule_name` / `metadata.description` |
| 触发条件        | 条件配置     | `metadata.conditions`                |
| 奖励模式        | 奖励配置     | `metadata.reward`                    |
| 阶梯表格        | 阶梯配置     | `metadata.reward.steps`              |
| 循环配置        | 循环扣除配置   | `metadata.reward.cycleMode`          |
| 上限控制        | 上限控制     | `metadata.reward.limits`             |
| 有效期         | 生效时间     | `effective_start` / `effective_end`  |
### 6.2 数据存储复用
AI 生成的规则直接存储到 `rule_definition` 表，与现有页面配置共用同一数据模型，无需新建表：

```sql
-- rule_definition 表（主设计文档 v7.3）
CREATE TABLE rule_definition (
    id BIGSERIAL PRIMARY KEY,
    program_code VARCHAR(32) NOT NULL,
    rule_code VARCHAR(64) NOT NULL,
    rule_name VARCHAR(200) NOT NULL,
    rule_type VARCHAR(30) NOT NULL,          -- DRL / ACTIVITY_PROMO
    rule_category VARCHAR(50) NOT NULL,      -- base / promo
    drl_content TEXT,
    status VARCHAR(20) DEFAULT 'DRAFT',
    version INT DEFAULT 1,
    metadata JSONB NOT NULL,
    effective_start TIMESTAMPTZ,
    effective_end TIMESTAMPTZ,
    rule_group VARCHAR(50),
    priority INT DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ,
    UNIQUE(program_code, rule_code)
);
```
### 6.3 前端组件复用
| AI 表单字段类型             | 复用页面组件          | 说明                    |
| --------------------- | --------------- | --------------------- |
| `condition_group`     | 积分活动页面 → 触发条件组件 | 实体选择 + 属性选择 + 运算符 + 值 |
| `step_table`          | 积分活动页面 → 阶梯规则表格 | 可增删行、拖拽排序             |
| `cycle_config`        | 积分活动页面 → 循环扣除配置 | 循环模式、循环点、剩余处理         |
| `limit_config`        | 积分活动页面 → 上限控制   | 单笔上限、累计上限、超限策略        |
| `point_type_selector` | 积分活动页面 → 积分类型下拉 | 按规则类型自动过滤             |
| `sku_selector`        | 商品选择组件          | 搜索、多选                 |

## 七、澄清对话模板
### 7.1 简单规则澄清模板

```text
用户：配置订单满100送10积分
AI：收到需求。请确认以下信息：
1. 这是长期生效的基础规则，还是指定时间段的限时活动？
2. 适用渠道是全部还是指定？
3. 是否限制会员等级？
用户：长期有效，全部渠道，全部会员。
AI：规则配置完成。订单实付金额≥100元，赠送10积分。长期有效。
```
### 7.2 指定商品多倍积分澄清模板

```text
用户：618活动指定商品多倍积分
AI：收到需求。在配置之前需要确认几个关键点：
1. 多倍积分是整单多倍还是商品多倍？
   A. 整单多倍：订单中包含指定商品即整单多倍
   B. 商品多倍：仅指定商品本身多倍，其他正常
2. 活动时间和时间基准是？
3. 多倍积分是多少倍？
用户：商品多倍。时间6月20-30日，付款时间判断。2倍。
AI：明白了。请选择参与活动的商品，并确认非指定商品的处理方式。
```
### 7.3 阶梯规则澄清模板

```text
用户：配置阶梯积分活动
AI：收到需求。请确认阶梯规则：
1. 阶梯计算的基准是什么？（订单金额 / 商品金额 / 其他）
2. 按金额区间不同倍数还是不同固定积分值？
3. 是否启用循环扣除？（如3000元以上分段循环）
用户：按订单金额，不同倍数，启用循环扣除。
AI：请填写阶梯区间表格：
   - 区间1：0-1000元 → 1倍
   - 区间2：1000-2000元 → 1.5倍
   - 区间3：2000-3000元 → 2倍
   - 区间4：3000元以上 → 3倍（循环点）
循环扣除方式：按3000元分段循环扣除。
```

## 八、总结
| 维度     | V3 设计   | V4 设计（本补充）                 |
| ------ | ------- | -------------------------- |
| 覆盖场景   | 仅简单积分规则 | 基础规则 + 活动规则全场景             |
| 奖励模式   | 仅固定倍数   | 固定倍数、固定积分值、阶梯倍数、阶梯固定值、阶梯循环 |
| 指定商品多倍 | 未明确区分   | **明确区分整单多倍 vs 商品多倍**       |
| 有效期    | 仅指定时间段  | **支持长期生效（不设置时间）**          |
| 现有页面复用 | 无       | **与积分活动页面、基础规则页面无缝集成**     |
| 数据存储   | 新建表     | **复用 `rule_definition` 表** |
| 阶梯配置   | 无       | 支持阶梯表格 + 循环配置              |

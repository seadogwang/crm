# Loyalty 主数据配置功能设计文档
> **关联主文档**：基于AI的下一代全渠道忠诚度管理SaaS平台 v7.3\
> **版本**：1.0\
> **设计目标**：为 Loyalty 系统提供统一的主数据管理能力，支持枚举值、层级数据、映射关系的可视化配置，使运营人员无需开发即可管理下拉选项、地址数据、商品映射等系统基础数据。
## 一、功能定位与设计理念
### 1.1 什么是“主数据配置”
主数据配置是管理系统中“静态字典数据”的功能，这些数据的特点是：
* **变化不频繁**：不会像积分规则那样频繁调整
* **被广泛引用**：在会员信息、订单数据、规则条件等多个模块中使用
* **需要业务人员可维护**：不应该让业务人员每次都要找开发去改
* **有层级或映射关系**：如省市区三级、SKU→产品名称的映射
### 1.2 与现有功能的关系
| 现有功能              | 与主数据配置的关系                               |
| ----------------- | --------------------------------------- |
| `program_schema`  | 字段定义中引用主数据（如 `gender` 字段引用 `GENDER` 枚举） |
| `rule_definition` | 规则条件中引用主数据（如 `gender == "MALE"`）        |
| `event_inbox`     | 事件数据中的值需要与主数据匹配（如 `channel` 字段）         |
| `member`          | 会员属性中的值来源于主数据                           |
### 1.3 设计原则
| 原则       | 说明                                    |
| -------- | ------------------------------------- |
| **统一管理** | 所有枚举值、层级数据、映射关系在同一个模块中管理              |
| **租户隔离** | 主数据按 `program_code` 隔离，支持 Program 级定制 |
| **可扩展**  | 支持新增自定义主数据类型，无需修改代码                   |
| **可追溯**  | 记录主数据变更历史，支持审计                        |
| **引用检测** | 删除主数据时检查是否被业务数据引用，防止数据不一致             |
## 二、整体架构
### 2.1 主数据类型分类
| 类型         | 说明             | 示例              | 管理方式 |
| ---------- | -------------- | --------------- | ---- |
| **枚举值**    | 固定选项列表，用于下拉选择  | 性别（男/女/未知）、订单状态 | 列表管理 |
| **层级数据**   | 树形结构，支持多级      | 省市区、商品分类        | 树形管理 |
| **映射表**    | Key-Value 映射关系 | SKU→产品名称        | 表格管理 |
| **标签/标签组** | 可打标的关键词        | 会员标签、商品标签       | 标签管理 |
### 2.2 数据模型
#### 2.2.1 主数据定义表
```sql
CREATE TABLE master_data_definition (
    id VARCHAR(64) PRIMARY KEY,
    program_code VARCHAR(32) NOT NULL,
    data_type VARCHAR(32) NOT NULL,          -- ENUM / HIERARCHY / MAPPING / TAG
    data_code VARCHAR(64) NOT NULL,          -- 唯一标识，如 GENDER / REGION / SKU_MAPPING
    data_name VARCHAR(128) NOT NULL,         -- 显示名称，如 "性别" / "省市区" / "SKU映射"
    description TEXT,
    config JSONB,                             -- 类型特有配置
    status VARCHAR(20) DEFAULT 'ACTIVE',     -- ACTIVE / INACTIVE
    created_by VARCHAR(64),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(program_code, data_code)
);
```
#### 2.2.2 枚举值表
```sql
CREATE TABLE master_data_enum (
    id VARCHAR(64) PRIMARY KEY,
    program_code VARCHAR(32) NOT NULL,
    data_code VARCHAR(64) NOT NULL,          -- 关联 master_data_definition
    enum_code VARCHAR(64) NOT NULL,          -- 枚举代码，如 "MALE"
    enum_label VARCHAR(128) NOT NULL,        -- 显示名称，如 "男"
    enum_value TEXT,                         -- 可选：对应的值
    sort_order INT DEFAULT 0,
    status VARCHAR(20) DEFAULT 'ACTIVE',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(program_code, data_code, enum_code)
);
```
#### 2.2.3 层级数据表
```sql
CREATE TABLE master_data_hierarchy (
    id VARCHAR(64) PRIMARY KEY,
    program_code VARCHAR(32) NOT NULL,
    data_code VARCHAR(64) NOT NULL,          -- 关联 master_data_definition
    node_code VARCHAR(64) NOT NULL,          -- 节点代码
    node_name VARCHAR(128) NOT NULL,         -- 节点名称
    parent_code VARCHAR(64),                 -- 父节点代码（根节点为 NULL）
    node_level INT DEFAULT 1,                -- 层级（1,2,3...）
    sort_order INT DEFAULT 0,
    status VARCHAR(20) DEFAULT 'ACTIVE',
    ext_attributes JSONB,                    -- 扩展属性
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(program_code, data_code, node_code)
);
```
#### 2.2.4 映射表数据
```sql
CREATE TABLE master_data_mapping (
    id VARCHAR(64) PRIMARY KEY,
    program_code VARCHAR(32) NOT NULL,
    data_code VARCHAR(64) NOT NULL,          -- 关联 master_data_definition
    key_code VARCHAR(128) NOT NULL,          -- 映射键
    key_label VARCHAR(256),                  -- 键的显示名称
    value_code VARCHAR(128) NOT NULL,        -- 映射值
    value_label VARCHAR(256),                -- 值的显示名称
    ext_attributes JSONB,                    -- 扩展属性
    status VARCHAR(20) DEFAULT 'ACTIVE',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(program_code, data_code, key_code)
);
```
#### 2.2.5 标签数据
```sql
CREATE TABLE master_data_tag (
    id VARCHAR(64) PRIMARY KEY,
    program_code VARCHAR(32) NOT NULL,
    tag_group VARCHAR(64),                   -- 标签组（可选）
    tag_code VARCHAR(64) NOT NULL,
    tag_name VARCHAR(128) NOT NULL,
    tag_color VARCHAR(16),                   -- 标签颜色
    status VARCHAR(20) DEFAULT 'ACTIVE',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(program_code, tag_code)
);
```
#### 2.2.6 变更历史表
```sql
CREATE TABLE master_data_audit (
    id BIGSERIAL PRIMARY KEY,
    program_code VARCHAR(32) NOT NULL,
    data_code VARCHAR(64) NOT NULL,
    operation VARCHAR(20),                   -- CREATE / UPDATE / DELETE
    old_value JSONB,
    new_value JSONB,
    operated_by VARCHAR(64),
    operated_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 六、总结
| 功能              | 说明                 |
| --------------- | ------------------ |
| **枚举值管理**       | 管理性别、订单状态、渠道等固定选项  |
| **层级数据管理**      | 管理省市区、商品分类等树形结构    |
| **映射表管理**       | 管理 SKU→产品名称等键值映射   |
| **标签管理**        | 管理会员标签、商品标签等       |
| **与规则引擎集成**     | 规则条件可引用主数据项        |
| **与 Schema 集成** | 字段定义可引用主数据类型       |
| **引用检查**        | 删除前检查被引用情况，防止数据不一致 |
